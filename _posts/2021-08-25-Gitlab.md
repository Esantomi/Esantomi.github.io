CI/CD 업무 때문에 GitLab CE를 쓸 일이 생겨 얼마 전 설치를 마쳤다. 참고차 Ubuntu 20.04 기준 GitLab CE 설치 방법을 정리해 두려 한다.

## GitLab? CE와 EE?

<img src="https://user-images.githubusercontent.com/61646760/130736429-cb8fb8f2-0a9a-4fff-8645-17a827da0b25.png" width="40%" height="40%"/>

**GitLab**은 기본적으로 GitLab Inc.에서 개발한 Wiki 및 Issue tracking 등의 기능을 갖춘 웹 기반 DevOps 시스템을 말한다. GitLab 라이선스는 **CE**와 **EE**로 나뉘어져 있는데, 전자는 **Community Edition**, 후자는 **Enterprise Edition**에 해당하며, EE는 유료 기능을 포함한다.

오늘은 GitLab CE를 설치해 볼 것이고, EE를 설치하는 과정도 별반 다르지 않으니 조금 서칭해 보면 되겠다.

## 설치 과정

1. 일단 시스템 업데이트를 한다. (참고로 `apt` 명령어로 `apt-get`을 대체 가능하다.)  
&#160;`sudo apt update`  
&#160;`sudo apt upgrade -y`

2. ssh 사용을 위한 ssh-server 설치  
&#160;`sudo apt install -y ca-certificates curl openssh-server`

3. GitLab 패키지 리포지토리 추가  
&#160;`curl -sS https://packages.gitlab.com/install/repositories/gitlab/gitlab-ce/script.deb.sh | sudo bash`

4. GitLab 설치  
&#160;`sudo apt -y install gitlab-ce`
![image](https://user-images.githubusercontent.com/61646760/130731490-415fd28f-6952-4407-90d2-c3ecead2d26f.png)

5. gitlab.rb 내부 external_url 설정  
&#160;`sudo vi /etc/gitlab/gitlab.rb`
- 해당 파일의 external_url 뒷부분 주소를 본인이 사용하고자 하는 로컬 주소로 변경해 준다. (vi 외의 에디터도 가능)
- 참고로 external_url 부분은 대략 28행 정도에 위치해 있다.
- 가령 `http://localhost:8001` 같은 식으로 변경해 주면 된다.

6. GitLab 초기 설정  
`sudo gitlab-ctl reconfigure`
![image](https://user-images.githubusercontent.com/61646760/130734493-07d07b53-7e14-4563-95bc-4a7e2d4a962c.png)

7. 본인이 설정한 로컬 주소로 접속
- 처음 접속하면 root 비밀번호를 설정하게 된다.
- 이후 id를 root로, 비밀번호는 설정한 값을 입력해 로그인하면 된다.
- 만약 비밀번호 설정을 까먹었다면? 이후 GitLab CE root 비밀번호 변경에 대해 포스팅해 보겠다. (내가 이랬다.)

8. 로그인하면 끝
- 이때 GitLab 아이디나 이메일 주소로 로그인을 하는 것이 아니다. 위에서 언급한 것처럼 root로 로그인을 하는 것이다.
