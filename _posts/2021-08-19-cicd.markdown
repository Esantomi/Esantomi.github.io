깃랩(GitLab)을 공부하려고 하니 데브옵스(DevOps), CI/CD 같은 개념들을 짚고 넘어가지 않을 수 없게 되었다. 그래서 이 참에 한번 정리해 보려고 한다.

## CI (Continuous Integration)

> 다수의 기여자(contributor)들에 의한 코드 변경을 하나의 소프트웨어 프로젝트로 자동적으로 통합(intergration)해 주는 작업

> the practice of automating the integration of code changes from multiple contributors into a single software project.

<p align="center">
  <img src="https://user-images.githubusercontent.com/61646760/130063610-174e963e-1dd4-4fa5-9378-9c46eedd01d0.png" />
</p>

DevOps의 주된 업무로, 이로써 개발자는 코드 변경 사항을 중앙 repository에 빈번하게 merge하고 build 및 test를 실행할 수 있다.

## CD (Continuous Delivery & Continuous Deployment) 

<p align="center">
  <img src="https://user-images.githubusercontent.com/61646760/130198167-4ced08b8-8c09-4ace-b766-76c44365e8c4.png" />
</p>

CD는 지속적인 서비스 제공(Continuous Delivery)과 지속적인 배포(Continuous Deployment)를 동시에 의미한다.

이때 **Continuous Delivery**는 변경 사항이 repository로 자동적으로 Release되는 것, **Continuous Deployment**는 고객이 사용 가능한 Production 환경까지 자동적으로 deploy되는 것을 뜻한다.

## 정리

아래 그림은 REDHAT에서 가져온 것으로 CI/CD를 한눈에 파악할 수 있게끔 도식화하고 있다.

- **Continuous Intergraion** : build - test - merge
- **Continuous Delivery** : Repository로 자동 릴리즈(release)
- **Continuous Deployment** : Production으로 자동 디플로이(deploy)
